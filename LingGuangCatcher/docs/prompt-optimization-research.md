## 二、当前 Prompt 评估

### 2.1 标签提取准确性

**现状**：标签由 AI 服务的 `callTextAPI` 硬编码传入，不是 AI 从内容中提取的。

```kotlin
val tags = listOf(noteType.tag, "语音")  // 固定：["随想", "语音"]
```

**问题**：
- 所有随想类笔记标签都是 `[随想, 语音]`，缺乏内容相关性
- 无法反映笔记的实际主题（如 "LifeOnline"、"测试"、"项目"）
- LifeOS 看板的标签筛选因此失去意义

**严重程度**：P1 — 影响检索和分类质量

---

### 2.2 内容整理的结构化程度

**现状**：THOUGHT 类型的 Prompt 要求"保持原有的思考脉络和情感，不要过度分析"，输出格式为：

```
## 💭 随想
[整理后的内容]
**关键词**
[3-5 个关键词]
```

**评估**：
- ✅ 保留了原始语音的情感和语气，文学性较强
- ✅ 去除了口语化词汇
- ⚠️ 对于极短的测试性语音（"我看一下能不能"），AI 过度扩写，生成了与原意不符的诗意内容
- ⚠️ 关键词格式不统一（有时用顿号、有时用逗号、有时用空格）

**严重程度**：P2 — 体验问题，不影响功能

---

### 2.3 GOAL 类型过度工程化

**现状**：`lingguang_milestone_2026-03-17_141344.md` 的原始语音是"我们看一下测试是否成功"（一句测试语音），但 AI 将其解析为一个完整的工程目标，生成了：
- ASR/NLU 置信度要求（≥95%）
- 熔断机制
- TTS 后校验环节
- 端到端耗时基准（≤3秒）

这是 GOAL 类型 Prompt 过于激进的问题——对任何输入都强制套用"里程碑 + 行动计划 + 成功标准 + 潜在障碍"模板。

**严重程度**：P1 — 对短句/测试性输入产生严重噪音

---

### 2.4 Frontmatter 中无多余字段

**现状**：所有样本的 Frontmatter 均符合协议，无多余字段。

**注意**：`lingguang_note_2026-03-17_141736.md` 被 LifeOS 看板操作后，Frontmatter 格式发生了变化（date 变为 ISO 格式、tags 变为多行 YAML），这是 LifeOS 写回时的格式差异，不是灵光 App 的问题。

## 三、优化方向

### 方向 A：AI 动态提取标签 [P1]

**目标**：让 AI 从内容中提取 2-4 个语义标签，替代当前的硬编码标签。

**方案**：在各 AI 服务的 Prompt 末尾增加指令：

```
最后，从内容中提取 2-4 个语义标签（不带 # 前缀），以 JSON 格式输出在正文末尾：
<!--tags: ["标签1", "标签2", "标签3"]-->
```

然后在 `CaptureRepository` 中解析这个注释，提取标签写入 Frontmatter。

**预期效果**：
- `lingguang_note_2026-03-17_133857.md` → `tags: [LifeOnline, 项目, 新开始]`
- `lingguang_note_2026-03-17_141736.md` → `tags: [测试, 验证, 期待]`

---

### 方向 B：短句保护机制 [P1]

**目标**：对极短的语音输入（< 15 字）降低 AI 扩写程度，避免过度诗意化或过度工程化。

**方案**：在 Prompt 中增加长度感知指令：

```
注意：如果原始内容少于 15 个字，请保持极简输出，不要扩写，只做基本整理。
```

---

### 方向 C：GOAL 类型 Prompt 收敛 [P1]

**目标**：GOAL 类型不强制要求"里程碑 + 行动计划 + 成功标准 + 潜在障碍"完整模板，改为按内容丰富度自适应输出。

**方案**：将 GOAL Prompt 改为：

```
如果内容足够具体，输出完整的目标规划结构；
如果内容较简短或模糊，只输出目标描述和 1-2 个初步行动项，不强制填充所有模板字段。
```

---

### 方向 D：关键词格式统一 [P2]

**目标**：统一关键词分隔符为逗号。

**方案**：在 Prompt 中明确：`关键词用逗号分隔，如：关键词1, 关键词2, 关键词3`

---

## 四、优先级排序

| 方向 | 优先级 | 预估工作量 | 影响范围 |
|------|--------|-----------|---------|
| A: AI 动态提取标签 | P1 | 1-2h | 所有 AI 服务 + CaptureRepository |
| B: 短句保护机制 | P1 | 0.5h | 所有 AI 服务 Prompt |
| C: GOAL 类型收敛 | P1 | 0.5h | DashScope/Gemini/OpenAI GOAL Prompt |
| D: 关键词格式统一 | P2 | 0.5h | 所有 AI 服务 Prompt |

**建议 Phase 3 优先实施 A + B + C**，合计约 2-3 小时工作量。

---

## 五、结论

当前 Prompt 体系的核心问题是**标签静态化**和**模板过度套用**。内容整理质量整体良好，文学性和可读性强，但在极短输入和特定类型（GOAL）上存在过度扩写问题。

建议 Phase 3 以"标签智能化 + 输入长度感知"为主线进行优化。
