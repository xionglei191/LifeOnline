Chapter 15

术语速查表：把整本书里的关键词真正变成你的语言

学软件时最让人头疼的，往往不是概念本身，而是术语太多。最后这一章的作用，是把高频词汇重新收束，让读者可以随时回来复习。

Chapter Guide

这一章的核心任务

把全书最常出现的词汇重新解释一遍，让它们不再是抽象名词，而成为你可以反复使用的基本语言。

01

基础技术词

Frontend 前端

用户直接看到和操作的那一层，负责呈现与交互。

Backend 后端

真正处理规则、权限、逻辑和数据的一层。

Database 数据库

系统长期保存数据的地方，可以理解为系统的记忆仓库。

API 接口

不同层之间交换信息的通道，前端通常通过 API 与后端通信。

02

工程与流程词

Requirement 需求

系统为什么存在、要解决什么问题、这一阶段先做什么。

Architecture 架构

系统如何分层、模块如何协作、数据如何流动。

Module 模块

系统中的一块相对独立功能区域，有清晰边界。

Testing 测试

用来确认功能正确、避免未来改坏的验证机制。

03

AI 时代高频词

LLM 大模型

Large Language Model，大语言模型。能理解和生成自然语言，也能辅助结构化工作。

Handoff Artifact 交接物

让下一轮人或 AI 能快速接上的正式产物，如任务书、图、测试结果、状态对象。

Verification 验证

确认 AI 产出是否真的正确，而不是只是“看起来像”。

Mainline 主线

当前阶段最值得优先推进的系统骨架，不应被低价值工作带跑。

04

LifeOnline 相关高频词

SoulAction

系统中的动作候选或行动对象，是从“理解”走向“行动”的关键中间层。

Governance 治理

对动作、权限、执行和结果进行审查、约束和管理的机制。

Reintegration 回流

把执行结果重新组织并纳入系统下一轮运行的过程。

Continuity 连续性

让高价值结果进入长期资产，形成可继承、可演进的系统记忆。

全书收束

如果这些词现在对你来说已经不再完全陌生，那说明这本书真正完成了一件事：它开始把你带进了“系统语言”里。
